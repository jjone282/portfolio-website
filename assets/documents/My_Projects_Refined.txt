Study Buddy Finder (Swift — SwiftUI, SwiftData, MapKit, CoreLocation)
Collaborative project (2 developers)
--------------------------------------------------------------------------------
Developed an iOS application enabling Arizona State University students to
discover and connect with nearby study partners in real time. The app follows
MVVM architecture, cleanly separating data models, business logic, and UI
across dedicated ViewModels.

Key features and technical highlights:
- Integrated CoreLocation and MapKit to render live session pins and the user's
  current location on an interactive campus map.
- Designed a session matching algorithm that filters active study sessions by
  shared course enrollment and geographic proximity.
- Consumed a university course catalog REST API via URLSession with async/await
  for dynamic, up-to-date course data loading.
- Persisted user profiles, course enrollments, and study session history
  across app launches using SwiftData.
- Implemented study session scheduling, post-session peer ratings, and local
  push notifications for upcoming sessions.
- Built structured list views using SwiftUI List with grouped sections for
  matched partners, active sessions, and notifications.

Park Explorer (Swift — SwiftUI, MapKit, MVVM)
--------------------------------------------------------------------------------
Architected a park discovery and management app following the MVVM design
pattern, separating park data structures, business logic, and SwiftUI views
into clean, maintainable layers without requiring CoreData or SwiftData.

Key features and technical highlights:
- Implemented a sectioned List view organizing parks alphabetically, with each
  row displaying the park's name, location, and thumbnail image.
- Preloaded five default parks on initialization, each with a name, location,
  image, and description, providing immediate content on first launch.
- Built dynamic Add and Delete functionality via an input form and
  swipe-to-delete gestures, with a fallback default image when no photo is
  provided.
- Integrated MapKit into the detail view to render an interactive map
  pinpointing each park's location by city.
- Implemented an in-map place search feature allowing users to query nearby
  points of interest (e.g., restaurants, coffee shops), rendering results as
  named annotations directly on the map.
- Utilized NavigationStack and NavigationLink for drill-down navigation from
  the park list into individual detail views.

My Travel Journal — Travel Logging App (Swift — SwiftUI, SwiftData, MVVM)
--------------------------------------------------------------------------------
Developed a persistent travel journal app enabling users to record, browse,
and manage visited cities, with all data retained across app sessions on-device.

Key features and technical highlights:
- Designed a City data model using SwiftData's @Model macro, storing each
  entry's name, photo, and description as persistently backed properties.
- Built an Add City flow with a PhotosPicker integration for selecting photos
  from the device library, with a default image applied when none is chosen.
- Leveraged SwiftData's @Query and @Environment(\.modelContext) for reactive,
  automatically updating list rendering whenever the data collection changes.
- Delegated all SwiftData query and mutation operations to a dedicated
  ViewModel layer, keeping SwiftUI views focused solely on presentation.
- Implemented swipe-to-delete functionality with automatic propagation of
  deletions to persistent storage.

L'Oréal Skincare Routine Builder (JavaScript, HTML/CSS, OpenAI API)
--------------------------------------------------------------------------------
Built an AI-powered web application enabling users to browse a catalog of real
L'Oréal skincare products, curate a personal selection, and receive a fully
personalized skincare or beauty routine generated by the OpenAI API.

Key features and technical highlights:
- Designed an interactive product grid with card-based selection and a
  real-time keyword search filter, allowing users to dynamically narrow the
  catalog as they type.
- Integrated the OpenAI API to generate personalized routines based on the
  user's selected products, passing structured JSON product data (name, brand,
  category, description) as context for generation.
- Built a conversational chat interface displaying AI-generated routines with
  support for context-aware follow-up questions, maintaining full conversation
  history for coherent multi-turn responses.
- Secured API key storage using a Cloudflare Worker to prevent client-side
  exposure.
- Persisted product selections across page reloads using localStorage.
- Implemented an RTL/LTR language toggle to support internationalization and
  right-to-left language layouts.
